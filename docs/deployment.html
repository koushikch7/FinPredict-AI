🐳 Docker Deployment (Recommended)

Option A: Pre-built Image

# Pull and run (when published to Docker Hub)
docker pull koushikch7/finpredict-ai:latest
docker run -d --name finpredict \
  -p 3000:3000 \
  --env-file .env \
  -v finpredict-data:/app/data \
  koushikch7/finpredict-ai:latest

Option B: Build from Source

# Clone repository
git clone https://github.com/koushikch7/FinPredict-AI.git
cd FinPredict-AI

# Create environment file
cp .env.example .env
nano .env  # Configure your settings

# Build image
docker build -t finpredict-ai .

# Run container
docker run -d --name finpredict \
  -p 3000:3000 \
  --env-file .env \
  -v finpredict-data:/app/data \
  --restart unless-stopped \
  finpredict-ai

Docker Compose

# docker-compose.yml
version: '3.8'
services:
  finpredict:
    build: .
    container_name: finpredict
    ports:
      - "3000:3000"
    env_file:
      - .env
    volumes:
      - finpredict-data:/app/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  finpredict-data:
# Launch with compose
docker compose up -d

⚙️ Environment Configuration

Required Variables

# Server
NODE_ENV=production
PORT=3000

# Security (generate with: openssl rand -hex 32)
JWT_SECRET=your-256-bit-secret-key
COOKIE_SECRET=your-256-bit-cookie-secret

# AI Provider (at least one required)
GEMINI_API_KEY=AIzaSy...your-key
# OR
ARBITER_API_KEY=gwtk_your_token
ARBITER_BASE_URL=https://your-arbiter-host/v1

Optional Variables

# Alternative AI providers
OPENAI_API_KEY=sk-...your-key

# FinBERT sentiment (HuggingFace)
HF_API_KEY=hf_...your-key

# Zerodha broker integration
KITE_API_KEY=your-kite-api-key
KITE_API_SECRET=your-kite-api-secret

# Backup configuration
BACKUP_ENABLED=true
BACKUP_RETENTION_DAYS=30

# Rate limiting
RATE_LIMIT_WINDOW_MS=60000
RATE_LIMIT_MAX_REQUESTS=100
⚠️ Security: Never commit .env to version control. Use secrets management in production.

🌐 Reverse Proxy Setup

Nginx Configuration

# /etc/nginx/sites-available/finpredict
server {
    listen 80;
    server_name finpredict.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}

# Enable the site
sudo ln -s /etc/nginx/sites-available/finpredict /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

SSL with Certbot

# Install certbot
sudo apt install certbot python3-certbot-nginx

# Get certificate
sudo certbot --nginx -d finpredict.yourdomain.com

# Auto-renewal is configured automatically

☁️ Cloud Deployment

AWS EC2

  1. Launch Ubuntu 22.04 t3.small instance

💾 Backup & Recovery

Automatic Backups

FinPredict automatically backs up the SQLite database daily when BACKUP_ENABLED=true:

  • Backups stored in /app/data/backups/
  • Filename format: backup-YYYY-MM-DD.db
  • Old backups purged after BACKUP_RETENTION_DAYS

Manual Backup

# Inside container
docker exec finpredict cp /app/data/finpredict.db /app/data/backups/manual-$(date +%Y%m%d).db

# Copy to host
docker cp finpredict:/app/data/backups/ ./backups/

Restore from Backup

# Stop container
docker stop finpredict

# Restore database
docker cp ./backups/backup-2026-06-15.db finpredict:/app/data/finpredict.db

# Start container
docker start finpredict

🛡️ Security Checklist

  • ✅ Use strong, unique secrets for JWT_SECRET and COOKIE_SECRET
  • ✅ Enable HTTPS with valid SSL certificate
  • ✅ Keep Docker and Node.js updated
  • ✅ Configure firewall (ufw) to only allow necessary ports
  • ✅ Use non-root user inside Docker container
  • ✅ Enable automatic security updates on host OS
  • ✅ Monitor logs for suspicious activity
  • ✅ Regular database backups with off-site copies
  • ✅ Use secrets manager (AWS Secrets Manager, Vault) in production

🆘 Troubleshooting

Container won't start

# Check logs
docker logs finpredict

# Common issues:
# - Missing required env vars → check .env file
# - Port already in use → change PORT or stop other service
# - Permission issues → check volume permissions

Database locked errors

# SQLite WAL mode should prevent this, but if it happens:
docker exec finpredict sqlite3 /app/data/finpredict.db "PRAGMA wal_checkpoint(TRUNCATE);"

Out of memory

# Increase container memory limit
docker run -d --name finpredict \
  -m 2g \  # 2GB limit
  --memory-swap 4g \
  ...
